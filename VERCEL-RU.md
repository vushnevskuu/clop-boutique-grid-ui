# Деплой витрины на Vercel (по-русски)

## Почему на сайте старый текст на английском

Vercel собирает проект **из последнего коммита в GitHub**, который привязан к проекту. Если правки есть только на компьютере и **не запушены**, на `*.vercel.app` остаётся **старая версия** (например, с Lovable и английским текстом).

**Что сделать:** закоммитьте и отправьте в GitHub все изменения ветки `main`, затем в Vercel нажмите **Redeploy** (или дождитесь автодеплоя).

```bash
cd /путь/к/витрине
git status
git add -A
git commit -m "Синхронизация витрины: русский UI, Medusa, Vercel"
git push origin main
```

## Если переименовали репозиторий на GitHub

Сейчас витрина живёт в репозитории **`vushnevskuu/c.lop_market_ru`** (имя с точкой — как на GitHub).

1. На GitHub: **Settings** репозитория — проверьте точное имя.
2. На Vercel: проект (например **clop-boutique-grid-ui**) → **Settings** → **Git**:
   - отключите и снова подключите репозиторий **с новым именем**,  
   - или проверьте, что Vercel подтянул переименование (иногда нужно **Reconnect**).
3. Убедитесь, что деплой идёт с ветки **`main`** и **Root Directory** = `/` (корень, если весь фронт в корне репо).

## Переменные окружения (Production)

**Settings** → **Environment Variables** → среда **Production** (при необходимости **Preview**):

| Имя | Значение |
|-----|----------|
| `VITE_MEDUSA_BACKEND_URL` | Публичный URL Medusa, например `https://xxxx.up.railway.app` (**без** `/app`) |
| `VITE_MEDUSA_PUBLISHABLE_KEY` | Ключ из админки Medusa → **Settings → Publishable API Keys** |

Сохраните → **Deployments** → последний деплой → **⋯** → **Redeploy** (обязательно: переменные `VITE_*` попадают в билд только при новой сборке).

Без Medusa в проде витрина может показывать каталог из `public/cloth/` или «товары не найдены» — это ожидаемо, пока бэкенд и ключ не настроены.

---

## Полезные кнопки в Vercel

- **Deployments** — история сборок, логи ошибок (**Build Logs**).
- **Settings → General** — имя домена, связь с Git.
- **Settings → Environment Variables** — секреты и URL для сборки.
- **Redeploy** — пересобрать после смены env или пуша.
